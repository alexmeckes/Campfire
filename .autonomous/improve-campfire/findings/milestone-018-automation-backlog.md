# milestone-018 automation backlog

The next Campfire backlog should focus on recurring Codex App automation patterns.

This fits the remaining roadmap and the repo priorities:

- recurring runs are the last major Codex App workflow surface still under-documented
- automation prompts need the same durable-state discipline as live rolling runs
- examples and docs can make recurring work safer without adding project-specific logic

## Proposed backlog

1. `milestone-018` - add a reusable automation-pattern reference for recurring Codex App runs
2. `milestone-019` - add deterministic verification or example coverage for automation-ready prompts and workspace guidance
3. `milestone-020` - document recurring automation patterns in README and example guidance

## Acceptance criteria

- Campfire provides a reusable reference for automation-ready prompts that pair with the task-state contract
- the repo contains deterministic evidence or example coverage for automation-ready workflow guidance
- README and self-hosted task state describe when to use recurring automations versus one-off rolling runs
